# Sandboxing as a Complement to Execution-Path Enforcement

<TagList tags={frontmatter.tags} />
<AttributionList contributors={frontmatter.contributors} />

Sandboxing alone does not solve AI security, but it is a necessary complement to execution-path enforcement. While
execution-path controls determine whether an action should occur, sandboxing constrains what happens if an action is
allowed. Together, these mechanisms ensure that AI systems remain safe even under partial failure or misclassification.
Sandboxing provides defense in depth by limiting the impact of mistakes, adversarial behavior, or unforeseen
interactions between modalities. In production, sandboxing should be treated as an always-on control rather than a
development-time feature, with continuous validation that isolation boundaries remain intact as systems evolve.
